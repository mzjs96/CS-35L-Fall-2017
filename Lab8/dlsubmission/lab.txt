Which of these calls are related to dynamic linking and what is the relationship?

The system calls that relates to the dynamic linking are:
mmap, access, open, fstat, read, mprotect


The system call access called the linker file ending with ld.so, it means that the following calls accessed the dynamic linker:

access: Chech the user's permission for the executable.
fstat: Obtain the information of the library, such as memory allocation.
mmap: Create a mapping for the library into the memory.
read: Read data information of library.
mprotect: Sets protection on the region of memory.
close: Close the library when the dynamic linking is done.
